Learn OpenFlow

Now you can learn the OpenFlow data plane using a simple web browser. Flowsim is designed to simulate five different versions of the OpenFlow switch data plane. Create a switch profile based on a specific version of OpenFlow. Instantiate that profile and configure the resulting switch as a live OpenFlow device. Step packets through each stage of the data plane of your switch, and see how the packet and your configuration interact at each stage in the pipeline.

Datapath

Constrain a switch's data plane fragmentation and packet buffering abilities through the datapath profile. Manipulate the data plane's fragmentation handling behaviors through the datapath configuration.

Flow Table

Determine a flow tables capabiltiess: number of flows, matching types, and instruction/action types. Track table lookups and hit/miss counts. Model a vendor's flow table with precise cababilities.

Flow

Experiment with capturing and manipulating packets through flow configuration. Track flow statisitcs and manage flow lifecycle explicit explusion, inactivity timeouts, and absolute timeouts.

Port

Model a vendor device with number of ports, medium, speeds, and capabilities. Toggle port status during simulation to see port state changes and the associated asynchronous OpenFlow events.

Match

Learn one of the key elements of OpenFlow by experimenting with fine and course grain matching. Explore the matching limitations of each version of OpenFlow and modeled vendor switches.

Instruction

Explore OpenFlow's control flow using goto's, partition matching across tables with metadata, and meter flow rates using policers and shapers. Learn immediate and delayed packet modification and forwarding.

Action

Mofify the contents of a packet, forward to a specific port, or with a unqiue egress handling group. Learn how immediate packet modification affects multi-table matching.

Group

Define a set of egress packet processing behaviors. Use Load balance across targets, select the live target from a high availability group, and copy a packet to multiple destinations.

Queue

Estblish port based queueing during profile creation. Understand OpenFlow's limitaitons on queue management, and use queues as targets in output actions.

Meter

Defing a policing or shaping strategy to meter a flow. Use this strategy to limit flows to target ceiling using strict discard, or allow for bursts up to a certain queue depth.
